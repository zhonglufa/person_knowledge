**本文档为提示词工程文档**

适用技术栈：**Vue2 + ElementUI + Vuex + axios**

用途：统一规范、自动生成可运行代码、数据库变更不崩溃、团队协作标准

**重要说明**：本系统 V2.2 版本**仅支持桌面端浏览器**，所有前端开发必须遵守桌面端-only策略。

------

# 0. 桌面端-only 策略（V2.2新增）

## 0.1 设备支持范围

| 设备类型 | 支持状态 | 说明 |
|---------|---------|------|
| **桌面浏览器** | ✅ 完全支持 | Chrome 90+、Firefox 88+、Edge 90+、Safari 14+ |
| **平板电脑** | ⚠️ 部分支持 | 可使用但未针对触摸操作优化 |
| **手机浏览器** | ❌ 不支持 | 不响应移动端布局 |

## 0.2 响应式样式规范

1. **禁止使用** `@media (max-width: 768px)` 移动端样式
2. **禁止使用** `@media (max-width: 480px)` 超小屏幕样式
3. **禁止使用** `.mobile-only`、`.hidden-mobile`、`.show-mobile` 等移动端工具类
4. **保留** `@media (min-width: 1024px)` 桌面端适配样式

## 0.3 组件开发规范

1. 所有组件默认按桌面端布局开发（最小宽度 1024px）
2. 不需要考虑触摸操作、滑动交互等移动端特性
3. 表格、表单、导航等组件按桌面端最佳实践设计
4. 图片、图标按桌面端分辨率优化（推荐 1920x1080）

## 0.4 删除的移动端代码

以下移动端相关代码已在 V2.2 版本中删除：

- ❌ 所有 `@media (max-width: 768px)` 响应式样式
- ❌ 移动端工具类：`.mobile-only`、`.hidden-mobile`、`.show-mobile`、`.hide-mobile`
- ❌ 响应式断点：`--breakpoint-xs: 480px`、`--breakpoint-sm: 768px`
- ❌ 移动端专属组件：汉堡菜单、抽屉导航、底部导航栏

# 1. 强制基础规则（AI 必须遵守）

1. 必须使用 **Vue2 语法**，禁止 Vue3
2. 必须使用 **ElementUI** 组件库
3. 必须使用 **Vuex** 状态管理
4. 必须使用 **axios** 统一请求
5. 所有页面必须 **容错、判空、默认值**
6. 取值必须使用 **可选链 a?.b** 或 **a&&a.b**
7. 数组默认值 `[]`，对象默认值 `{}`
8. 接口必须 `try catch` + `loading`
9. 方法名必须 **动词开头**：get/add/update/delete/open
10. 列表渲染 `v-for` 必须带 `key`
11. 禁止 `v-for` 与 `v-if` 同节点
12. 样式必须加 `scoped`

- 页面 / 组件：**大驼峰** Home.vue、CollectionCenter.vue

# 5. axios 封装规范（utils/request.js）

```
import axios from 'axios'
import { Message } from 'element-ui'

const service = axios.create({
  baseURL: '/api',
  timeout: 10000
})

// 请求拦截
service.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = 'Bearer ' + token
  }
  return config
})

// 响应拦截
service.interceptors.response.use(res => {
  return res.data
}, err => {
  Message.error('请求失败：' + err.message)
  return Promise.reject(err)
})

export default service
```

# 7. 数据库变更不崩溃规则（核心）

1. 对象取值必须判空

```
item?.title || ''
user && user.nickname
```

1. 数组默认空数组

```
this.list = res?.data || []
```

1. 接口必须容错

```
async getList() {
  try {
    this.loading = true
    const res = await getCollectionList()
    this.list = res?.data || []
  } catch (e) {
    this.$message.error('加载失败')
  } finally {
    this.loading = false
  }
}
```

1. 字段必须做映射，不直接渲染后端字段

# 8. 路由规范（router/index.js）

```
import Vue from 'vue'
import Router from 'vue-router'
import Home from '@/views/Home.vue'
import Login from '@/views/Login.vue'

Vue.use(Router)

const routes = [
  { path: '/', component: Home },
  { path: '/login', component: Login },
  {
    path: '/collection',
    component: () => import('@/views/CollectionCenter.vue'),
    meta: { requireAuth: true }
  },
  {
    path: '/creation',
    component: () => import('@/views/CreationCenter.vue'),
    meta: { requireAuth: true }
  },
  {
    path: '/personal',
    component: () => import('@/views/PersonalCenter.vue'),
    meta: { requireAuth: true }
  },
  {
    path: '/search',
    component: () => import('@/views/SearchCenter.vue'),
    meta: { requireAuth: true }
  }
]

const router = new Router({ routes })

// 路由守卫
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (to.meta.requireAuth && !token) {
    next('/login')
  } else {
    next()
  }
})

export default router
```

# 9. Vuex 规范（store/modules/user.js）

```java
const state = {
  userInfo: {},
  token: localStorage.getItem('token') || ''
}

const mutations = {
  SET_USER_INFO(state, info) {
    state.userInfo = info || {}
  },
  SET_TOKEN(state, token) {
    state.token = token
    localStorage.setItem('token', token)
  },
  LOGOUT(state) {
    state.userInfo = {}
    state.token = ''
    localStorage.removeItem('token')
  }
}

const actions = {
  async getUserInfo({ commit }) {
    // 调用接口获取信息
  }
}

export default {
  namespaced: true,
  state,
  mutations,
  actions
}
```
