## 1. 管理员端

### 1.1 用户管理模块

#### 功能描述
支持管理员查看全平台用户列表，按用户名、邮箱、角色、状态筛选，执行禁用/解封操作，后台新增和编辑用户信息。严格权限隔离，管理员不可查看或修改用户私有内容。

#### 对应接口清单表

| # | 方法 | 路径 | 请求参数 | 认证 | 功能描述 |
|---|------|------|---------|------|---------|
| 62 | GET | `/admin/users` | Query: `pageNum`, `pageSize`, `keyword`, `role` | 管理员 | 获取全平台用户列表（分页+筛选） |
| 63 | PUT | `/admin/users/{id}/disable` | Path: `id` | 管理员 | 禁用用户（无法登录） |
| 64 | PUT | `/admin/users/{id}/enable` | Path: `id` | 管理员 | 解封用户 |
| 95 | POST | `/admin/users` | Body: `username`, `email`, `password`, `nickname`(可选), `role` | 管理员 | 后台新增用户（无需验证码） |
| 96 | PUT | `/admin/users/{id}` | Path: `id`; Body: `nickname`, `role` | 管理员 | 后台编辑用户信息 |
| 113* | PUT | `/admin/users/batch-disable` | Body: `userIds[]`, `reason` | 管理员 | **[增强]** 批量禁用用户 |
| 113* | PUT | `/admin/users/batch-enable` | Body: `userIds[]` | 管理员 | **[增强]** 批量启用用户 |
| 114* | GET | `/admin/users/export` | Query: 同列表查询 | 管理员 | **[增强]** 导出用户列表（Excel） |

#### 功能拆解：管理用户状态（禁用/解封）

**对应接口**：`PUT /admin/users/{id}/disable` 和 `PUT /admin/users/{id}/enable`

**请求参数**：
- Path:
  - `id` (Long, 必填) - 用户ID
- Body: 无

**响应数据结构**：
```json
{
  "code": 200,
  "message": "用户状态已更新",
  "data": {
    "userId": 1001,
    "status": 0,
    "updatedAt": "2026-04-27 10:30:00"
  }
}
```

**前端逻辑**：
1. 用户管理页 `UserManagement.vue` 渲染用户列表表格，包含"操作"列
2. 点击"禁用"按钮，触发 `handleDisableUser(row)` 方法
3. 弹出 `ElMessageBox.confirm` 二次确认模态框，提示"确定要禁用该用户吗？"
4. 确认后调用 `userManageApi.disableUser(row.id)` 发起 PUT 请求
5. 请求成功：
   - 更新 `userList` 中对应项的 `status` 字段为 0
   - 显示 `ElMessage.success('禁用成功')`
   - 刷新分页列表 `loadUserList()`
6. 请求失败：显示 `ElMessage.error('操作失败')`，保持原状态
7. "解封"按钮逻辑类似，调用 `userManageApi.enableUser(row.id)`

**数据流向过程**：
1. **前端**：管理员点击禁用按钮 → Vue 组件调用 `userManageApi.disableUser(id)`
2. **HTTP 请求**：`PUT /admin/users/{id}/disable`，携带 JWT Token（管理员角色）
3. **后端控制器**：`AdminController.disableUser()` 接收请求
   - 校验管理员权限（`@RequireAdmin` 注解）
   - 提取路径参数 `id`
4. **服务层**：`AdminServiceImpl.disableUser()` 执行业务逻辑
   - 查询 `users` 表：`SELECT id, username, status FROM users WHERE id = ?`
   - 校验用户是否存在、是否已禁用
   - 更新 `users.status = 0`（0=禁用，1=正常）
   - 记录操作日志到 `admin_operation_log` 表：
     ```sql
     INSERT INTO admin_operation_log (operator_id, action, target_type, target_id, detail, created_at)
     VALUES (?, 'disable_user', 'user', ?, '{"reason":"违规操作"}', NOW())
     ```
   - 清除该用户的登录态（Redis 中删除对应 token）
5. **数据库**：执行 `UPDATE users SET status = 0, updated_at = NOW() WHERE id = ?`
6. **响应返回**：返回标准 `{code: 200, message: "用户已禁用", data: {...}}`
7. **前端更新**：接收响应，更新 UI 状态，刷新列表

### 1.2 内容管理与审核模块

#### 功能描述
支持管理员查看全平台公开内容（收藏集、笔记），按内容类型筛选，对违规内容执行下架/恢复操作。下架后内容转为私有状态，不可被其他用户查看。所有操作记录日志到 `admin_operation_log` 表，可回溯。

#### 对应接口清单表

| # | 方法 | 路径 | 请求参数 | 认证 | 功能描述 |
|---|------|------|---------|------|---------|
| 65 | GET | `/admin/content` | Query: `pageNum`, `pageSize`, `contentType`, `searchKey`, `isPublic`(可选) | 管理员 | 获取全平台公开内容列表 |
| 66 | PUT | `/admin/content/take-down` | Body: `contentType`, `contentId` | 管理员 | 下架公开内容（转为私有状态） |
| 107 | PUT | `/admin/content/restore` | Body: `contentType`, `contentId` | 管理员 | 恢复已下架内容 |
| 108 | GET | `/admin/content/audit-logs` | Query: `pageNum`, `pageSize`, `operationType`, `targetType`, `startTime`, `endTime` | 管理员 | 内容操作日志查询 |
| 115* | POST | `/admin/content/batch-take-down` | Body: `contentType`, `contentIds[]`, `reason` | 管理员 | **[增强]** 批量下架内容 |
| 115* | POST | `/admin/content/batch-restore` | Body: `contentType`, `contentIds[]` | 管理员 | **[增强]** 批量恢复内容 |

#### 功能拆解：下架违规内容

**对应接口**：`PUT /admin/content/take-down`

**请求参数**：
- Body:
  - `contentType` (string, 必填) - 内容类型，`collection` 或 `note`
  - `contentId` (Long, 必填) - 内容ID

**响应数据结构**：
```json
{
  "code": 200,
  "message": "内容已下架",
  "data": {
    "contentId": 5001,
    "contentType": "collection",
    "isPublic": 0,
    "operatedAt": "2026-04-27 11:00:00"
  }
}
```

**前端逻辑**：
1. 内容管理页 `ContentManagement.vue` 展示公开内容列表（Tab 切换收藏集/笔记）
2. 每行操作列显示"下架"按钮（`isPublic === 1` 时显示）
3. 点击"下架"按钮，触发 `handleTakeDownContent(type, row)` 方法
4. 弹出 `ElMessageBox.confirm` 确认框："确定要下架该收藏集吗？下架后其他用户将无法查看。"
5. 确认后调用 `contentManageApi.takeDownContent({ contentType: type, contentId: row.id })`
6. 请求成功：
   - 更新列表项 `row.isPublic = 0`
   - 按钮变为"恢复"（条件渲染 `v-if="scope.row.isPublic === 0"`）
   - 显示 `ElMessage.success('下架成功')`
7. 请求失败：显示错误提示，保持原状态

**数据流向过程**：
1. **前端**：管理员点击下架按钮 → 调用 `contentManageApi.takeDownContent()`
2. **HTTP 请求**：`PUT /admin/content/take-down`，Body: `{ contentType: 'collection', contentId: 5001 }`
3. **后端控制器**：`AdminController.takeDownContent()` 接收请求
   - 校验管理员权限
   - 解析 Body 参数
4. **服务层**：`AdminServiceImpl.takeDownContent()` 执行事务操作（`@Transactional`）
   - 根据 `contentType` 判断操作表：
     - `collection` → 操作 `collection` 表
     - `note` → 操作 `note` 表
   - 查询内容是否存在且为公开状态：
     ```sql
     SELECT id, user_id, is_public FROM collection WHERE id = ? AND is_public = 1
     ```
   - 更新内容公开状态：
     ```sql
     UPDATE collection SET is_public = 0, updated_at = NOW() WHERE id = ?
     ```
   - 若为收藏集，同步下架其下所有收藏项：
     ```sql
     UPDATE collection_item SET is_public = 0 WHERE collection_id = ?
     ```
   - 记录操作日志到 `admin_operation_log` 表：
     ```sql
     INSERT INTO admin_operation_log (operator_id, action, target_type, target_id, detail, created_at)
     VALUES (?, 'take_down_content', 'collection', ?, '{"reason":"违规内容"}', NOW())
     ```
5. **数据库**：执行上述 SQL（事务包裹，保证一致性）
6. **响应返回**：返回 `{code: 200, message: "内容已下架", data: {...}}`
7. **前端更新**：接收响应，更新 UI 状态，按钮切换为"恢复"

#### 功能描述

| 70 | PUT | `/admin/announcements/{id}` | Path: `id`; Body: 同创建 | 管理员 | 编辑公告 |
| 71 | DELETE | `/admin/announcements/{id}` | Path: `id` | 管理员 | 下架公告（已下架状态可物理删除） |

#### 功能拆解：发布公告（触发全站通知）

**对应接口**：`POST /admin/announcements/{id}/publish`

**请求参数**：
- Path:
  - `id` (Long, 必填) - 公告ID
- Body: 无

**响应数据结构**：
```json
{
  "code": 200,
  "message": "公告已发布，通知已推送给所有用户",
  "data": {
    "announcementId": 3001,
    "status": "published",
    "publishedAt": "2026-04-27 14:00:00",
    "notifyCount": 1500
  }
}
```

**前端逻辑**：
1. 公告管理页 `AnnouncementManagement.vue` 展示公告列表
2. 草稿状态公告（`status === 0`）操作列显示"发布"按钮
3. 点击"发布"按钮，触发 `handlePublishAnnouncement(row)` 方法
4. 弹出 `ElMessageBox.confirm` 确认框："确定要发布公告吗？发布后将推送给全平台用户。"
5. 确认后调用 `announcementApi.publishAnnouncement(row.id)`
6. 请求成功：
   - 更新列表项 `row.status = 1`（已发布）
   - 按钮变为"下架"和"统计"
   - 显示 `ElMessage.success('公告已发布')`
   - 刷新列表
7. 请求失败：显示错误提示

**数据流向过程**：
1. **前端**：管理员点击发布按钮 → 调用 `announcementApi.publishAnnouncement(id)`
2. **HTTP 请求**：`POST /admin/announcements/{id}/publish`
3. **后端控制器**：`AdminController.publishAnnouncement()` 接收请求
4. **服务层**：`AdminServiceImpl.publishAnnouncement()` 执行事务操作
   - 查询公告是否存在且为草稿状态：
     ```sql
     SELECT id, title, content FROM announcement WHERE id = ? AND status = 'draft'
     ```
   - 更新公告状态：
     ```sql
     UPDATE announcement SET status = 'published', published_at = NOW(), updated_at = NOW() WHERE id = ?
     ```
   - 查询全平台所有用户（排除管理员）：
     ```sql
     SELECT id, username FROM users WHERE role = 'commonUser' AND status = 1
     ```
   - 批量插入通知记录到 `notification` 表（异步操作 `@Async`）：
     ```sql
     INSERT INTO notification (user_id, notify_type, title, content, is_read, created_at)
     VALUES (?, 2, ?, ?, 0, NOW())
     ```
   - 记录操作日志到 `admin_operation_log` 表
5. **数据库**：执行上述 SQL（事务包裹）
6. **响应返回**：返回 `{code: 200, message: "公告已发布", data: {notifyCount: 1500}}`
7. **前端更新**：接收响应，显示推送成功提示，刷新列表

| 111 | GET | `/admin/dashboard/trends` | Query: `days`, `dataType` | 管理员 | 获取趋势数据 |

#### 功能拆解：获取后台首页统计数据

**对应接口**：`GET /admin/dashboard`

**请求参数**：无

**响应数据结构**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userStats": {
      "totalUsers": 1500,
      "todayNewUsers": 25,
      "activeUsers7d": 800
    },
    "contentStats": {
      "totalCollections": 3000,
      "totalItems": 15000,
      "totalNotes": 5000,
      "publicContents": 2000
    },
    "interactionStats": {
      "totalLikes": 10000,
      "totalComments": 3000,
      "totalCollects": 8000
    }
  }
}
```

**前端逻辑**：
1. 仪表盘页 `Home.vue` 加载时调用 `loadDashboardData()` 方法
2. 调用 `dashboardApi.getDashboardStats()` 发起 GET 请求
3. 请求成功：
   - 更新 `dashboardData` 状态对象
   - 渲染统计卡片（用户统计、内容统计、互动统计）
   - 渲染趋势图（Canvas 绘制）
   - 渲染系统健康状态卡片
4. 启动定时器 `setInterval(loadDashboardData, 30000)` 每30秒刷新
5. 组件销毁时清除定时器 `clearInterval()`

**数据流向过程**：
1. **前端**：页面加载 → 调用 `dashboardApi.getDashboardStats()`
2. **HTTP 请求**：`GET /admin/dashboard`
3. **后端控制器**：`AdminController.getDashboard()` 接收请求
4. **服务层**：`AdminServiceImpl.getDashboard()` 执行统计查询
   - 用户统计：
     ```sql
     SELECT COUNT(*) as totalUsers FROM users WHERE deleted = 0
     SELECT COUNT(*) as todayNewUsers FROM users WHERE DATE(created_at) = CURDATE()
     SELECT COUNT(DISTINCT user_id) as activeUsers7d FROM users WHERE last_login_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
     ```
   - 内容统计：
     ```sql
     SELECT COUNT(*) as totalCollections FROM collection WHERE deleted = 0
     SELECT COUNT(*) as totalItems FROM collection_item WHERE deleted = 0
     SELECT COUNT(*) as totalNotes FROM note WHERE deleted = 0
     SELECT COUNT(*) as publicContents FROM collection WHERE is_public = 1 AND deleted = 0
     UNION ALL SELECT COUNT(*) FROM note WHERE is_public = 1 AND deleted = 0
     ```
   - 互动统计：
     ```sql
     SELECT COUNT(*) as totalLikes FROM interaction_like WHERE deleted = 0
     SELECT COUNT(*) as totalComments FROM interaction_comment WHERE deleted = 0
     SELECT COUNT(*) as totalCollects FROM interaction_collect WHERE deleted = 0
     ```
5. **数据库**：执行聚合查询（使用索引优化）
6. **响应返回**：组装统计数据返回
7. **前端更新**：接收数据，渲染统计卡片和趋势图

## 2. 用户端

### 2.1 用户认证与个人管理模块

| 76 | PUT | `/user/settings` | Body: `theme`, `notifyPreferences`, `displayMode` | 是 | 更新个性化设置 |

#### 功能拆解：用户登录

**对应接口**：`POST /user/login`

**请求参数**：
- Body:
  - `username` (string, 必填) - 用户名或邮箱
  - `password` (string, 必填) - 密码（明文，后端加密校验）

**响应数据结构**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userId": 1001,
    "username": "testuser",
    "email": "test@example.com",
    "role": "commonUser",
    "avatar": "https://oss.example.com/avatar/1001.jpg"
  }
}
```

**前端逻辑**：
1. 登录页 `Login.vue` 渲染表单（用户名/邮箱输入框、密码输入框、登录按钮）
2. 用户输入账号密码，点击"登录"按钮
3. 触发 `handleLogin()` 方法，执行表单验证
4. 调用 `authApi.login({ username, password })` 发起 POST 请求
5. 请求成功：
   - 存储 Token 到 `localStorage`：`localStorage.setItem('token', response.data.token)`
   - 存储用户信息到 Vuex：`this.$store.commit('user/SET_USER_INFO', response.data)`
   - 根据角色跳转：
     - `role === 'admin'` → `router.push('/admin/home')`
     - 否则 → `router.push('/personal/dashboard')`
   - 显示 `ElMessage.success('登录成功')`
6. 请求失败：
   - 显示错误提示（用户名不存在、密码错误、账号已禁用）

| # | 方法 | 路径 | 请求参数 | 认证 | 功能描述 |
|---|------|------|---------|------|---------|
| 75 | GET | `/user/settings` | 无 | 是 | 获取用户个性化配置 |
| 76 | PUT | `/user/settings` | Body: `theme`, `notifyPreferences`, `displayMode` | 是 | 更新个性化设置 |

#### 功能拆解：更新个性化设置

**对应接口**：`PUT /user/settings`

**请求参数**：
- Body:
  - `theme` (string, 可选) - 主题，`light` 或 `dark`
  - `notifyPreferences` (object, 可选) - 通知偏好配置（JSON格式）
  - `displayMode` (string, 可选) - 展示模式，`card` 或 `list`

**响应数据结构**：
```json
{
  "code": 200,
  "message": "设置已保存",
  "data": {
    "theme": "dark",
    "notifyPreferences": {
      "learningRemind": true,
      "systemAnnouncement": true,
      "noteRemind": false,
      "interaction": true
    },
    "displayMode": "card"
  }
}
```

**前端逻辑**：
1. 设置页 `Settings.vue` 渲染配置表单
2. 用户修改主题（Switch 切换）、推送开关（Checkbox 组）、展示模式（Radio）
3. 修改触发 `handleUpdateSettings()` 方法（防抖 500ms）
4. 调用 `userApi.updateSettings(formData)`
5. 请求成功：
   - 更新 Vuex 中的 `userSettings` 状态
   - 应用主题：`document.body.classList.toggle('dark', theme === 'dark')`
   - 更新列表展示模式
   - 显示 `ElMessage.success('设置已保存')`
6. 请求失败：恢复原设置值，显示错误提示

**数据流向过程**：
1. **前端**：用户修改设置 → 调用 `userApi.updateSettings()`
2. **HTTP 请求**：`PUT /user/settings`，Body: `{ theme: "dark", notifyPreferences: {...} }`
3. **后端控制器**：`UserController.updateSettings()` 接收请求
4. **服务层**：`UserServiceImpl.updateSettings()` 执行业务逻辑
   - 查询用户设置记录：
     ```sql
     SELECT id, user_id FROM user_settings WHERE user_id = ?
     ```
   - 若不存在，创建默认记录：
     ```sql
     INSERT INTO user_settings (user_id, theme, notify_preferences, display_mode)
     VALUES (?, 'light', '{"learningRemind":true,"systemAnnouncement":true}', 'card')
     ```
   - 更新设置：
     ```sql
     UPDATE user_settings SET theme = ?, notify_preferences = ?, display_mode = ?, updated_at = NOW()
     WHERE user_id = ?
     ```
5. **数据库**：执行 `UPDATE` 或 `INSERT` 语句
6. **响应返回**：返回 `{code: 200, message: "设置已保存", data: {...}}`
7. **前端更新**：接收数据，应用主题和展示模式
