## 二、产品概述

### 2.1 产品名称

个人结构化知识系统

### 2.2 产品定位

面向个人用户的全链路知识管理工具，核心帮助用户完成 **「碎片化知识采集→结构化加工拆解→体系化分类管理→深度笔记沉淀→学习进度闭环→全局检索复用」** 的完整业务闭环，将零散的网页、资料、多媒体内容，转化为可管理、可学习、可检索、可复用的个人专属结构化知识库。

### 2.3 产品核心目标

1. 降低个人知识采集门槛，支持浏览器收藏夹批量导入，一键完成碎片化内容收录；
2. 提供标准化的知识结构化加工能力，通过分类、标签、摘要、关键词等维度，实现零散内容的体系化管理；
3. 打通知识采集到深度沉淀的链路，支持收藏内容关联笔记创作，完成知识的内化吸收；
4. 搭建学习进度闭环，通过消化状态、进度标记、定时提醒，实现知识学习的全流程跟进；
5. 基于 ES 引擎实现毫秒级全局检索，让用户积累的个人知识库可快速复用，解决知识沉淀后找不到、用不上的痛点。

### 2.5 设备支持

**重要说明**：本版本（V2.2）**仅支持桌面端浏览器**，不支持移动端设备。

| 设备类型 | 支持状态 | 说明 |
|---------|---------|------|
| **桌面浏览器** | ✅ 完全支持 | Chrome 90+、Firefox 88+、Edge 90+、Safari 14+ |
| **平板电脑** | ⚠️ 部分支持 | 可使用但未针对触摸操作优化 |
| **手机浏览器** | ❌ 不支持 | 不响应移动端布局，建议使用桌面设备访问 |

**设计决策**：
1. 专注桌面端用户体验，避免多端适配导致的开发成本增加
2. 知识管理系统核心场景为深度学习和内容创作，桌面端更符合使用习惯
3. 删除所有 `@media (max-width: 768px)` 响应式样式，保持代码整洁

## 三、用户角色与权限说明

本系统核心分为 2 类用户角色，角色权限与数据库`users`表`role`字段完全对应，具体规则如下：

|  角色名称  |  角色标识  |              核心定位              |                         核心权限范围                         |
| :--------: | :--------: | :--------------------------------: | :----------------------------------------------------------: |
|  普通用户  | commonUser | 系统核心使用者、个人知识库的所有者 | 1. 账号注册、登录、个人信息管理全权限2. 个人收藏集、收藏项的增删改查全权限3. 个人分类、标签的创建与管理权限4. 个人笔记的创作、编辑、分享、收藏全权限5. 个人学习进度、提醒通知的管理权限6. 系统全局检索、个人内容检索全权限 |
| 系统管理员 |   admin    |   系统运维、内容监管、公告发布者   | 1. 全平台用户账号管理权限（查看、禁用、解封）2. 全平台公开内容审核、管理权限3. 系统公告发布、全平台通知推送权限4. 系统运营数据查看、统计权限5. 无普通用户的个人内容操作权限 |

**核心规则**：所有用户数据严格按`user_id`隔离，普通用户仅可操作、查看自己的个人数据，管理员不可查看、修改普通用户的私有内容，仅可对违规公开内容进行下架处理。

## 四、核心业务流程

### 4.1 核心主流程：知识采集到复用全链路

1. 用户登录系统
2. 知识采集：手动添加收藏项 / 批量导入浏览器收藏夹
3. 结构化加工：给内容分配分类、绑定标签、提炼摘要与关键词
4. 深度沉淀：基于收藏内容创作关联笔记，完成知识内化
5. 学习闭环：设置消化状态、学习进度、定时提醒，跟进学习
6. 检索复用：通过全局检索快速找到对应知识，实现内容复用
7. 复用过程中可再次进行结构化加工，形成闭环

### 4.2 辅助流程 1：个人中心操作流程

1. 用户登录
2. 进入个人中心
3. 分支操作：
   - 查看 / 修改个人基础信息、上传头像
   - 查看个人知识数据统计
   - 查看 / 管理系统通知消息
   - 个性化系统设置

### 4.3 辅助流程 2：全局检索流程

1. 用户进入检索中心
2. 输入检索关键词
3. ES 引擎执行全库检索，按 user_id 做权限隔离
4. 返回匹配结果，高亮关键词
5. 用户通过多维度筛选条件过滤结果
6. 用户点击结果进入详情页，完成内容查看 / 编辑
7. 系统自动记录检索历史，更新热门检索词统计

## 五、详细功能需求说明

### 5.1 账号与权限模块

#### 5.1.1 用户注册

- **功能描述**：支持新用户通过用户名 + 邮箱完成账号注册，创建个人专属知识库
- **前置条件**：用户未登录系统，访问注册页面
- **业务规则**：
  - [ ] 用户名、邮箱为必填项，全局唯一，不可重复注册
  - [ ] 密码需满足 8-20 位，包含字母 + 数字组合，加密存储
  - [ ] 注册成功后，自动创建普通用户角色账号，生成默认分类与收藏集
  - [ ] 注册成功后，自动跳转登录页面
- **输入项**：用户名、邮箱、密码、确认密码
- **输出项**：注册成功 / 失败提示，失败需明确说明原因（用户名已存在、邮箱已注册等）
- **后置条件**：系统生成用户账号数据，写入`users`表，默认角色为 commonUser

#### 5.1.2 用户登录

- **功能描述**：支持已注册用户通过用户名 / 邮箱 + 密码完成登录，进入系统
- **前置条件**：用户已完成注册，账号未被禁用
- **业务规则**：
  - [ ] 支持用户名 / 邮箱两种登录账号方式
  - [ ] 密码错误超过 5 次，临时锁定账号 10 分钟，避免暴力破解
  - [ ] 登录成功后，生成登录态 token，记录用户最后登录时间
  - [ ] 管理员账号登录后，自动跳转后台管理页面，普通用户跳转系统首页
- **输入项**：登录账号（用户名 / 邮箱）、密码
- **输出项**：登录成功 / 失败提示，失败需明确说明原因
- **后置条件**：用户获取系统操作权限，更新`users`表`last_login_at`字段

#### 5.1.3 产品首页（游客访问）

- **功能描述**：未登录用户访问产品首页，展示产品价值和优质内容，引导用户注册登录
- **前置条件**：用户未登录，访问 `/guest/home` 路由
- **业务规则**：
  - [ ] 产品首页为纯公开宣传页，无需登录态即可访问
  - [ ] 展示产品核心价值主张、功能特色介绍
  - [ ] **产品首页导航限制**：仅展示Logo和登录/注册按钮，禁止显示系统功能导航项（如收藏中心、创作中心、检索中心等）
  - [ ] **热门知识区域**：展示系统推荐的公开笔记（Top 10）
    - 筛选条件：`isPublic=true AND status='published' AND digestStatus='digested'`
    - 排序规则：热度评分 `hotScore = views*0.6 + likes*0.4` 降序
    - 展示内容：笔记标题、描述、封面图、浏览量、点赞数
  - [ ] **推荐收藏集区域**：展示系统推荐的公开收藏集（Top 10）
    - 数据来源：`/collections/recommended` 接口
    - 展示内容：收藏集名称、描述、封面图、收藏数
  - [ ] 点击热门知识/推荐收藏集，引导用户登录查看详情
  - [ ] **登录/注册入口交互**：
    - 点击"免费注册"按钮：跳转到 `/login?tab=register`（使用query参数控制显示注册表单）
    - 点击"登录"按钮：跳转到 `/login`（默认显示登录表单）
    - 注册成功后：自动切换到登录表单（同一页面），用户名自动填充到登录表单
    - 登录/注册表单支持Tab切换，切换时自动清除表单校验信息
  - [ ] **表单校验交互规范**：
    - 所有表单字段添加 `trigger: 'blur'`，失焦时触发校验
    - 实时反馈：输入错误时显示提示，用户修正时立即清除错误
    - 非顺序填写：每个字段独立校验，互不干扰
  - [ ] API 失败时静默降级，不显示错误页面，仅隐藏对应区域
- **输入项**：无（匿名访问）
- **输出项**：产品宣传内容、热门知识列表、推荐收藏集列表
- **后置条件**：无

**接口需求**：

| 接口编号 | 方法 | 接口路径 | 功能描述 | 请求参数 |
|---------|------|---------|---------|---------|
| 新增* | GET | `/note/recommended` | 获取推荐笔记（匿名） | Query: `pageNum`, `pageSize` |
| 83 | GET | `/collections/recommended` | 获取推荐收藏集（匿名） | Query: `pageNum`, `pageSize` |

**数据库变更**：

| 表名 | 字段名 | 类型 | 说明 |
|------|--------|------|------|
| `note` | `views` | INT UNSIGNED | 浏览量，默认 0 |
| `note` | `likes` | INT UNSIGNED | 点赞数，默认 0 |
| `note` | `hot_score` | DECIMAL(10,2) | 热度评分 = views*0.6 + likes*0.4 |
| `note` | `digest_status` | VARCHAR(20) | 消化状态：undigest/digesting/digested（与collection_item保持一致） |

#### 5.1.4 忘记密码

- **功能描述**：支持用户通过注册邮箱完成密码重置，找回账号
- **前置条件**：用户已注册账号，访问忘记密码页面
- **业务规则**：
  - [ ] 输入注册邮箱，发送验证码，验证码有效期 15 分钟
  - [ ] 验证通过后，可设置新密码，新密码不可与原密码一致
  - [ ] 密码重置成功后，自动清空原有登录态，需重新登录
- **输入项**：注册邮箱、邮箱验证码、新密码、确认新密码
- **输出项**：重置成功 / 失败提示
- **后置条件**：更新`users`表密码字段，原有登录态失效

### 5.2 个人中心模块

#### 5.2.1 个人基础信息管理

- **功能描述**：支持用户查看、修改个人基础信息，上传 / 更换头像
- **前置条件**：用户已成功登录系统
- **业务规则**：
  - [ ] 可查看信息：用户名、注册邮箱、昵称、头像、注册时间、用户角色
  - [ ] 可编辑信息：昵称、头像、绑定邮箱（需验证）、登录密码
  - [ ] 用户名不可修改，注册邮箱修改需通过新邮箱验证码验证
  - [ ] 头像支持 JPG/PNG 格式，大小不超过 2MB，上传后自动生成缩略图
  - [ ] 密码修改需验证原密码，修改成功后自动退出登录，需重新登录
- **输入项**：昵称、头像文件、新邮箱、邮箱验证码、原密码、新密码
- **输出项**：信息修改成功 / 失败提示，头像实时更新展示
- **后置条件**：更新`users`表对应用户的信息字段

#### 5.2.2 个人统计信息展示

- **功能描述**：可视化展示用户个人知识库的核心数据统计，直观呈现知识积累情况
- **前置条件**：用户已成功登录系统
- **业务规则**：
  - [ ] 核心统计指标：收藏集总数、收藏项总数、已消化内容数、创作笔记总数、未读通知数
  - [ ] 趋势统计：近 7 天新增收藏项、新增笔记数量趋势图
  - [ ] 分类统计：各分类、各收藏集的内容数量占比
  - [ ] 学习进度统计：未消化、消化中、已消化、已放弃内容的数量占比
  - [ ] 所有统计数据仅展示当前登录用户的个人数据，不可查看其他用户数据
- **输入项**：无
- **输出项**：可视化统计卡片、趋势图、占比饼图
- **后置条件**：无

#### 5.2.3 通知消息管理

- **功能描述**：展示用户所有系统通知，支持已读 / 未读管理、删除操作
- **前置条件**：用户已成功登录系统
- **业务规则**：
  - [ ] 通知分类：收藏项学习提醒、学习进度到期提醒、笔记提醒、系统公告
  - [ ] 列表按时间倒序排列，未读通知置顶展示，高亮标记
  - [ ] 支持单条标记已读、一键全部标记已读
  - [ ] 支持单条通知软删除，删除后不在列表展示，不可恢复
  - [ ] 点击通知可直接跳转至对应业务页面（如收藏项详情、笔记详情）
- **输入项**：标记已读、删除操作指令
- **输出项**：通知列表、操作成功 / 失败提示
- **后置条件**：更新`notification`表`is_read`、`read_at`、`deleted`字段

#### 5.2.4 系统个性化设置

- **功能描述**：支持用户自定义系统个性化配置，适配个人使用习惯
- **前置条件**：用户已成功登录系统
- **业务规则**：
  - [ ] 支持系统主题切换：浅色主题（light）、深色主题（dark），默认浅色
  - [ ] 支持消息推送开关：可单独开启/关闭收藏项学习提醒、系统公告、笔记提醒、互动通知等各类通知的推送提醒，推送偏好以 JSON 格式存储
  - [ ] 支持列表展示方式切换：列表模式（list）、卡片模式（card），默认卡片模式

#### 5.9.5 互动数据安全与限制

- **功能描述**：保障互动功能的安全性、合规性，防止滥用与违规内容传播
- **前置条件**：用户进行点赞、评论、收藏操作
- **业务规则**：
  - [ ] 评论字数限制：1-500 字，超出拦截并提示
  - [ ] 敏感词过滤：评论内容经过敏感词库匹配，命中则拦截并提示
  - [ ] 防刷机制：单用户单内容点赞/收藏操作间隔不低于 1 秒，评论操作间隔不低于 5 秒
  - [ ] 违规评论处理：用户可举报违规评论，管理员审核后下架违规评论
  - [ ] 内容下架联动：目标内容被下架后，该内容的点赞、评论数据仍保留，但不在前端展示
  - [ ] 隐私保护：互动数据严格按 user_id 隔离，用户仅可查看自己的互动记录
- **输入项**：互动操作请求
- **输出项**：操作成功/失败提示（含拦截原因）
- **后置条件**：记录互动操作日志，违规操作触发告警通知

## 六、非功能需求

### 6.1 性能需求

1. 页面加载性能：所有前端页面首屏加载时间≤2s，静态资源加载完成时间≤3s；
2. 接口响应性能：普通业务接口响应时间≤500ms，复杂统计查询接口响应时间≤1s；
3. 检索性能：基于 ES 的全局检索响应时间≤1s，带高级筛选的检索响应时间≤2s；
4. 并发性能：系统支持同时在线用户数≥1000 人，单接口并发量≥100QPS，无报错、无数据错乱；
5. 批量操作性能：浏览器收藏夹导入支持单次最多 1000 条链接，导入完成时间≤10s。

### 6.2 安全性需求

1. 数据加密：用户密码采用不可逆加密算法存储，不可明文存储、不可解密；
2. 权限隔离：严格按 user_id 实现用户数据隔离，接口需做权限校验，禁止越权访问、修改其他用户数据；
3. 接口安全：所有接口需做参数校验、防 SQL 注入、防 XSS 攻击，敏感操作需做二次验证；
4. 登录安全：支持密码错误次数限制、账号临时锁定，防止暴力破解；
5. 数据安全：支持数据库定时备份，每日全量备份、每小时增量备份，备份数据保留 30 天，可快速恢复；
6. 合规要求：用户数据严格遵守个人信息保护相关法规，不泄露、不滥用用户个人数据与私有内容。

### 6.3 兼容性需求

1. 浏览器兼容：支持 Chrome、Edge、Firefox、Safari 等主流浏览器最新 2 个版本，无样式错乱、无功能异常；
2. 设备兼容：PC 端适配 1920*1080、1366*768 等主流分辨率，移动端适配手机、平板等设备，页面自适应展示；
3. 系统兼容：支持 Windows、macOS、iOS、Android 等主流操作系统，无功能异常。

### 6.4 易用性需求

1. 操作流程极简：核心功能操作步骤不超过 3 步，用户可快速上手，无需复杂学习；
2. 引导清晰：新用户首次登录提供功能引导，操作异常时给出明确的错误提示与解决方案；
3. 容错性：支持操作撤销、草稿自动保存，防止用户误操作导致数据丢失；
4. 一致性：系统所有页面的 UI 风格、交互逻辑、按钮位置保持一致，降低用户学习成本。

### 6.5 可扩展性需求

1. 功能扩展性：系统架构支持后续新增知识导入渠道、AI 加工、知识图谱等新功能，无需重构核心架构；
2. 检索扩展性：ES 检索引擎支持后续新增检索字段、检索范围，可快速适配新的业务内容；
3. 容量扩展性：支持用户数据量持续增长，单用户收藏项数量≥10 万条时，系统性能无明显下降。

### 7.2 核心设计规范

1. 所有表必须包含主键 id，采用 bigint 自增类型，唯一标识每条数据；
2. 所有用户相关表必须包含 user_id 字段，关联 users.id，实现用户数据隔离；
3. 所有业务表必须包含软删除字段 deleted，tinyint 类型，0 = 未删除，1 = 已删除，默认 0；
4. 所有表必须包含创建时间、更新时间字段，自动生成、自动更新，便于数据回溯；
5. 所有关联字段必须添加外键约束，保证数据一致性，避免脏数据；
6. 所有高频查询字段必须添加索引，提升查询性能，避免全表扫描。

## 八、核心技术方案建议

### 8.1 整体技术架构

推荐采用前后端分离的架构设计，具体选型建议如下：

|  架构层  |          推荐技术选型          |                           选型说明                           |
| :------: | :----------------------------: | :----------------------------------------------------------: |
|   前端   | Vue2.7.16 + element-ui:2.15.14 |       轻量化、高性能，适配 PC 端，组件丰富，开发效率高       |
|   后端   |   Spring Boot + MyBatis-Plus   | 成熟稳定的 Java 生态，适配 MySQL 数据库，开发便捷，扩展性强  |
|  数据库  |           MySQL 8.0            |  与已提供的数据库设计完全兼容，性能稳定，支持事务、外键约束  |
| 检索引擎 |      Elasticsearch 7.17+       | 行业标准全文检索引擎，完美适配系统全局检索需求，支持分词、高亮、高性能检索 |
| 文件存储 | 本地存储 / MinIO / 阿里云 OSS  |   适配头像、封面、附件等文件的存储，可根据部署规模灵活选择   |

### 8.2 ES 检索引擎核心方案

1. **数据同步方案**：采用 MySQL+Canal+ES 的实时同步方案，MySQL 业务数据发生新增、修改、删除时，通过 Canal 监听 binlog 日志，实时同步至 ES 索引，保证检索数据与业务数据一致性；
2. **索引设计**：按内容类型创建独立索引，分为收藏项索引、笔记索引、收藏集索引，每个索引包含核心检索字段、权限字段（user_id）、筛选字段，便于分范围检索与筛选；
3. **分词方案**：采用 IK 分词器，支持中文智能分词，同时配置自定义词库，提升专业知识内容的检索匹配度；
4. **权限控制**：所有检索请求必须携带 user_id，ES 检索时强制过滤 user_id，确保用户仅可检索自己的私有内容 + 公开内容，避免越权访问。

### 8.3 浏览器收藏夹导入方案

1. 前端解析用户上传的 HTML 收藏夹文件，通过 DOM 解析提取文件夹结构、网页标题、链接地址；
2. 前端生成预览列表，用户勾选需要导入的内容后，将数据提交至后端；
3. 后端先做去重校验，过滤当前用户已存在的相同 URL 收藏项；
4. 校验通过后，批量创建收藏集、收藏项数据，写入对应数据库表；
5. 导入完成后，返回导入结果，包含成功数量、失败数量、失败原因。
